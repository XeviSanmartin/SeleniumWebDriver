# MultipleWindowsTest.java - Múltiples finestres i pestanyes

---
Quan una acció obre una nova finestra o pestanya del navegador, Selenium no canvia el focus automàticament — continua controlant la finestra original. Per interactuar amb la nova finestra cal canviar explícitament el focus amb `driver.switchTo().window(handle)`. Aquest test treballa contra [the-internet.herokuapp.com/windows](http://the-internet.herokuapp.com/windows).

___

### 📄 Codi del test — dues versions

#### Versió 1 — Simple (per índex d'array)

```csharp
@Test
public void multipleWindows() {
    driver.get("http://the-internet.herokuapp.com/windows");

    // Fer clic a l'enllaç que obre una nova finestra
    driver.findElement(By.cssSelector(".example a")).click();

    // Obtenir tots els handles de finestra com a array
    Object[] allWindows = driver.getWindowHandles().toArray();

    // Canviar a la finestra original (índex 0) i verificar
    driver.switchTo().window(allWindows[0].toString());
    assertThat(driver.getTitle(), is(not("New Window")));

    // Canviar a la nova finestra (índex 1) i verificar
    driver.switchTo().window(allWindows[1].toString());
    assertThat(driver.getTitle(), is("New Window"));
}
```

#### Versió 2 — Robusta (per comparació de handles)

```typescript
MultipleWindowsTest.java — Múltiples finestres i pestanyes@Test
public void multipleWindowsRedux() {
    driver.get("http://the-internet.herokuapp.com/windows");

    // Guardar el handle de la finestra actual abans d'obrir la nova
    String firstWindow = driver.getWindowHandle();
    String newWindow = "";

    // Fer clic a l'enllaç que obre la nova finestra
    driver.findElement(By.cssSelector(".example a")).click();

    // Obtenir tots els handles
    Set<String> allWindows = driver.getWindowHandles();

    // Identificar el handle de la nova finestra per descart
    for (String window : allWindows) {
        if (!window.equals(firstWindow)) {
            newWindow = window;
        }
    }

    // Canviar a la finestra original i verificar
    driver.switchTo().window(firstWindow);
    assertThat(driver.getTitle(), is(not(equalTo("New Window"))));

    // Canviar a la nova finestra i verificar
    driver.switchTo().window(newWindow);
    assertThat(driver.getTitle(), is(equalTo("New Window")));
}
```

___

### 🔍 Conceptes clau

#### Què és un _window handle_?

Cada finestra o pestanya oberta pel navegador té un identificador únic anomenat **window handle** — una cadena de text opaca que Selenium utilitza per referir-se a aquella finestra concreta.

```javascript
// Handle de la finestra actualment activa
String handle = driver.getWindowHandle(); // Ex: "CDwindow-4B5C3A..."

// Set amb els handles de TOTES les finestres obertes
Set<String> handles = driver.getWindowHandles();
```

___

#### Diferència entre les dues versions

|  | Versió 1 (per índex) | Versió 2 (per comparació) ✅ |
|---------------------------------|-------------------------------------|---------------------------------------------------|
| **Com identifica la nova finestra** | Per posició a l'array (`[0]`, `[1]`) | Comparant amb el handle original |
| **Fiabilitat** | ⚠️ L'ordre del Set no està garantit | ✅ Sempre identifica correctament la nova finestra |
| **Escalabilitat** | Només funciona bé amb 2 finestres | Funciona amb qualsevol nombre de finestres |
| **Ús recomanat** | Demos ràpides | Tests de producció |

### 🧰 Operacions amb finestres — referència completa

```javascript
// Obtenir el handle de la finestra activa
String finestraActual = driver.getWindowHandle();

// Obtenir tots els handles oberts
Set<String> totesLesFinestres = driver.getWindowHandles();

// Canviar el focus a una finestra concreta
driver.switchTo().window(handle);

// Tancar la finestra activa (sense tancar el driver)
driver.close();

// Tornar a la finestra original després de tancar la nova
driver.switchTo().window(finestraActual);

// Obrir una nova pestanya buida (Selenium 4)
driver.switchTo().newWindow(WindowType.TAB);

// Obrir una nova finestra buida (Selenium 4)
driver.switchTo().newWindow(WindowType.WINDOW);
```

> ⚠️ **Important:** `driver.close()` tanca només la finestra activa. Si tanques l'última finestra oberta sense cridar `driver.quit()`, el procés del driver quedarà orfe a la memòria. Utilitza sempre `driver.quit()` al `@AfterClass` per tancar-ho tot de forma neta.
