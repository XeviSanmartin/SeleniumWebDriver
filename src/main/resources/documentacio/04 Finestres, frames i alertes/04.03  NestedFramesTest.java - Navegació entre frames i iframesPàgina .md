# NestedFramesTest.java - Navegació entre frames i iframes | Odissea

---
Els `<frame>` i `<iframe>` són documents HTML encastats dins d'un altre document HTML. Selenium tracta cada frame com un context de navegació independent: per interactuar amb elements dins d'un frame cal **canviar el focus** a aquell frame primer amb `driver.switchTo().frame()`. Aquest test treballa contra [the-internet.herokuapp.com/nested\_frames](https://the-internet.herokuapp.com/nested_frames).

___

### 🗺️ Estructura de frames de la pàgina

```scss
  Pàgina principal (defaultContent)
  ├── frame-top
  │   ├── frame-left
  │   ├── frame-middle
  │   └── frame-right
  └── frame-bottom
```

La pàgina té un frame superior (`frame-top`) que conté tres frames aniuats (left, middle, right), i un frame inferior (`frame-bottom`) al mateix nivell.

___

### 📄 Codi del test

```csharp
NestedFramesTest.java — Navegació entre frames i iframespublic class NestedFramesTest extends BaseTest {

    // Mètode d'utilitat: llista tots els frames de la pàgina actual
    public void listIframesFromPage(WebDriver driver) {
        final List<WebElement> iframes = driver.findElements(By.tagName("frame"));
        for (WebElement iframe : iframes) {
            System.out.println(iframe.getAttribute("id"));
            System.out.println(iframe.getAttribute("name"));
        }
    }

    @Test
    public void testApp() {
        driver.navigate().to("https://the-internet.herokuapp.com/nested_frames");

        // 1. Llistar els frames del document principal
        listIframesFromPage(driver);

        // 2. Entrar a frame-bottom i llegir el seu contingut
        driver.switchTo().frame("frame-bottom");
        System.out.println(driver.findElement(By.xpath("html/body")).getText());

        // 3. Tornar al document principal
        driver.switchTo().defaultContent();

        // 4. Entrar a frame-top
        driver.switchTo().frame("frame-top");

        // 5. Llistar els frames aniuats dins de frame-top
        listIframesFromPage(driver);

        // 6. Des de frame-top, entrar a frame-middle i llegir el seu contingut
        driver.switchTo().frame("frame-middle");
        System.out.println(driver.findElement(By.xpath("html/body")).getText());
    }
}
```

### 🔍 Anàlisi pas a pas

#### Pas 1 — Llistar frames del document principal

```scss
listIframesFromPage(driver);
// Imprimeix els id i name de: frame-top i frame-bottom
```

El mètode d'utilitat usa `findElements(By.tagName("frame"))` per obtenir tots els frames del context actual i imprimeix els seus atributs `id` i `name`. Molt útil per explorar l'estructura d'una pàgina desconeguda.

#### Pas 2 — Entrar a frame-bottom

```less
driver.switchTo().frame("frame-bottom");
System.out.println(driver.findElement(By.xpath("html/body")).getText());
// Output: "BOTTOM"
```

Un cop dins del frame, el context de Selenium és el document d'aquell frame. Qualsevol `findElement()` cercarà _dins del frame_, no a la pàgina principal.

#### Pas 3 — Tornar al document principal

```scss
driver.switchTo().defaultContent();
```

`defaultContent()` retorna el focus al document arrel de la pàgina, independentment de quants nivells de frames s'hagin aniuat. És el "reset" del context de frames.

#### Passos 4, 5 i 6 — Frames aniuats

```scss
driver.switchTo().frame("frame-top");    // Entrar a frame-top
listIframesFromPage(driver);              // Veure frames aniuats: left, middle, right
driver.switchTo().frame("frame-middle"); // Entrar a frame-middle (des de frame-top)
System.out.println(driver.findElement(By.xpath("html/body")).getText());
// Output: "MIDDLE"
```

Per accedir a un frame aniuat cal entrar primer al frame pare i després fer un segon `switchTo().frame()` per al frame fill. **No es pot saltar directament** al frame fill des del document principal.

___

### 🧰 Les tres formes de fer `switchTo().frame()`

| Forma | Codi | Quan usar-la |
|----------------|--------------------------------------|-------------------------------------------------|
| **Per nom o id** | `driver.switchTo().frame("frame-top")` | El frame té atribut `name` o `id` estable ✅ |
| **Per índex** | `driver.switchTo().frame(0)` | El frame no té nom ni id (fràgil, no recomanat) |
| **Per WebElement** | `driver.switchTo().frame(element)` | Quan localitzes el frame amb `findElement()` ✅ |

```java
// Per nom o id (més comú)
driver.switchTo().frame("frame-middle");

// Per índex (0 = primer frame de la pàgina)
driver.switchTo().frame(0);

// Per WebElement
WebElement elMeuFrame = driver.findElement(By.id("frame-middle"));
driver.switchTo().frame(elMeuFrame);
```

### 🗺️ Navegació entre contextos — referència completa

```scss
// Entrar en un frame
driver.switchTo().frame("nom-o-id");

// Pujar UN nivell (del frame fill al frame pare)
driver.switchTo().parentFrame();

// Tornar al document arrel (des de qualsevol nivell)
driver.switchTo().defaultContent();
```

| Mètode | Efecte | Des d'on |
|-----------------------------|------------------------------|----------------------------------|
| `switchTo().frame(ref)` | Entra al frame indicat | Document arrel o frame pare |
| `switchTo().parentFrame()` | Puja un nivell al frame pare | Des d'un frame fill |
| `switchTo().defaultContent()` | Torna al document arrel | Des de qualsevol nivell de frame |

> 💡 **Regla d'or:** Quan acabis de treballar dins d'un frame, crida sempre `driver.switchTo().defaultContent()` abans de continuar amb la resta del test. En cas contrari, els següents `findElement()` cercaran dins del frame i no trobaran els elements de la pàgina principal.
