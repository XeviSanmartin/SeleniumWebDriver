# AlertsTest.java - Gestió d'alertes JavaScript 

---
Les alertes JavaScript són finestres emergents natives del navegador que bloquegen la interacció amb la pàgina fins que l'usuari les tanca. Selenium les gestiona a través de la interfície `Alert` i el mètode `driver.switchTo().alert()`. Aquest test treballa contra [the-internet.herokuapp.com/javascript\_alerts](https://the-internet.herokuapp.com/javascript_alerts).

___

### 🧩 Els tres tipus d'alerta JavaScript

| Tipus | Descripció | Botons | Accepta text? |
|---------|----------------------------|-------------|---------------|
| **Alert** | Missatge informatiu simple | OK | No |
| **Confirm** | Pregunta de confirmació | OK / Cancel | No |
| **Prompt** | Sol·licita text a l'usuari | OK / Cancel | ✅ Sí |

___

### 📄 Codi del test

```php
AlertsTest.java — Gestió d'alertes JavaScriptpublic class AlertsTest extends BaseTest {

    // --- Mètodes d'utilitat reutilitzables ---

    public void acceptAlert() {
        try {
            WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(2));
            wait.until(ExpectedConditions.alertIsPresent());
            Alert alert = driver.switchTo().alert();
            alert.accept(); // Prem OK
        } catch (Exception e) { /* gestió d'excepció */ }
    }

    public void dismissAlert() {
        try {
            WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(2));
            wait.until(ExpectedConditions.alertIsPresent());
            Alert alert = driver.switchTo().alert();
            alert.dismiss(); // Prem Cancel
        } catch (Exception e) { /* gestió d'excepció */ }
    }

    public void promptAlert() {
        try {
            WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(2));
            wait.until(ExpectedConditions.alertIsPresent());
            Alert alert = driver.switchTo().alert();
            alert.sendKeys("Hola hola super hola"); // Escriu text al prompt
            alert.accept(); // Confirma
        } catch (Exception e) { /* gestió d'excepció */ }
    }

    // --- Test principal ---

    @Test
    public void testApp() throws InterruptedException {
        driver.navigate().to("https://the-internet.herokuapp.com/javascript_alerts");

        // 1. Alert simple → acceptar amb OK
        driver.findElement(By.xpath(".//*[@id='content']/div/ul/li[1]/button")).click();
        Thread.sleep(2000);
        acceptAlert();
        Thread.sleep(2000);

        // 2. Confirm → acceptar amb OK
        driver.findElement(By.xpath(".//*[@id='content']/div/ul/li[2]/button")).click();
        Thread.sleep(2000);
        acceptAlert();
        Thread.sleep(2000);

        // 3. Confirm → cancel·lar amb Dismiss
        driver.findElement(By.xpath(".//*[@id='content']/div/ul/li[2]/button")).click();
        Thread.sleep(2000);
        dismissAlert();
        Thread.sleep(2000);

        // 4. Prompt → escriure text i acceptar
        driver.findElement(By.xpath(".//*[@id='content']/div/ul/li[3]/button")).click();
        Thread.sleep(2000);
        promptAlert();
        Thread.sleep(2000);
    }
}
```

### 🔍 Anàlisi: els tres mètodes d'utilitat

#### Patró comú als tres mètodes

Els tres mètodes segueixen el mateix patró de tres passos:

1.  **Esperar** que l'alerta aparegui amb `WebDriverWait + alertIsPresent()`
2.  **Canviar el focus** a l'alerta amb `driver.switchTo().alert()`
3.  **Interactuar** amb ella: `accept()`, `dismiss()` o `sendKeys()`

```java
// Patró base per a qualsevol alerta
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(2));
wait.until(ExpectedConditions.alertIsPresent()); // Esperar que aparegui
Alert alert = driver.switchTo().alert();          // Canviar focus a l'alerta
```

___

#### `alert.accept()` — Prémer OK

```bash
Alert alert = driver.switchTo().alert();
alert.accept(); // Equivalent a prémer el botó "OK" o "Acceptar"
```

Vàlid per als tres tipus d'alerta (Alert, Confirm i Prompt).

#### `alert.dismiss()` — Prémer Cancel

```bash
Alert alert = driver.switchTo().alert();
alert.dismiss(); // Equivalent a prémer "Cancel" o tancar l'alerta amb la X
```

En una alerta simple (_Alert_), `dismiss()` té el mateix efecte que `accept()` perquè només hi ha un botó. En _Confirm_ i _Prompt_ cancel·la l'operació.

#### `alert.sendKeys()` — Escriure en un Prompt

```java
Alert alert = driver.switchTo().alert();
alert.sendKeys("text a introduir"); // Escriu al camp del prompt
alert.accept();                      // Confirma
```

Només té efecte en alertes de tipus **Prompt**. En Alert i Confirm s'ignora.

#### `alert.getText()` — Llegir el missatge de l'alerta

```java
Alert alert = driver.switchTo().alert();
String missatge = alert.getText();
System.out.println(missatge); // "I am a JS Alert"
```

___

### 📊 Resum de mètodes de la interfície `Alert`

| Mètode | Acció | Tipus d'alerta |
|----------------------|--------------------------------------|------------------------|
| `alert.accept()` | Prem el botó OK / Acceptar | Alert, Confirm, Prompt |
| `alert.dismiss()` | Prem el botó Cancel / Tanca l'alerta | Alert, Confirm, Prompt |
| `alert.sendKeys(text)` | Escriu text al camp d'entrada | Només Prompt |
| `alert.getText()` | Llegeix el missatge de l'alerta | Alert, Confirm, Prompt |

> ⚠️ **Important:** Si hi ha una alerta oberta i Selenium intenta interactuar amb qualsevol element de la pàgina, llançarà una `UnhandledAlertException`. Sempre cal gestionar l'alerta abans de continuar amb el test.
