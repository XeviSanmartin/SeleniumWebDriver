# Estratègies de localització d'elements | Odissea

---
Abans de poder interactuar amb qualsevol element d'una pàgina web, Selenium necessita **trobar-lo**. Per a això disposa de diverses estratègies de localització, cadascuna adequada a un cas d'ús diferent.

En Java, totes s'utilitzen a través de la classe `By`:

```bash
WebElement element = driver.findElement(By.id("elMeuId"));
```

___

### 🔎 Les 6 estratègies principals

#### 1\. `By.id()` — Per atribut `id`

L'estratègia més ràpida i fiable. Utilitza l'atribut `id` de l'element HTML.

```python
<input id="search_form_input_homepage" type="text" />
```

```bash
WebElement camp = driver.findElement(By.id("search_form_input_homepage"));
```

> ✅ **Recomanat sempre que l'element tingui un id únic i estable.**

___

#### 2\. `By.name()` — Per atribut `name`

Útil en formularis, on els camps solen tenir l'atribut `name`.

```lua
<input name="username" type="text" />
```

```java
WebElement camp = driver.findElement(By.name("username"));
```

___

#### 3\. `By.cssSelector()` — Per selector CSS

Molt potent i flexible. Utilitza la mateixa sintaxi que CSS per estilitzar pàgines. És l'estratègia més utilitzada en projectes professionals.

```bash
<button class="btn btn-primary" id="login-btn">Entrar</button>
```

```java
// Per classe CSS
WebElement btn = driver.findElement(By.cssSelector(".btn-primary"));

// Per id
WebElement btn = driver.findElement(By.cssSelector("#login-btn"));

// Per atribut
WebElement btn = driver.findElement(By.cssSelector("button[type='submit']"));

// Combinat: div que conté un input
WebElement camp = driver.findElement(By.cssSelector("div.form-group input"));
```

___

#### 4\. `By.xpath()` — Per expressió XPath

El més potent de tots. Permet navegar per l'estructura del DOM amb gran precisió. Útil quan no hi ha `id` ni classes fiables.

```php-template
<div class="results">
  <span>Resultat 1</span>
</div>
```

```css
// Ruta absoluta (fràgil, no recomanada)
driver.findElement(By.xpath("/html/body/div/span"));

// Ruta relativa (recomanada)
driver.findElement(By.xpath("//div[@class='results']/span"));

// Per text visible
driver.findElement(By.xpath("//span[text()='Resultat 1']"));

// Per atribut parcial
driver.findElement(By.xpath("//input[contains(@id,'search')]"));
```

> ⚠️ **Precaució:** els XPath absoluts són molt fràgils davant de canvis en l'HTML. Utilitza sempre rutes relatives amb atributs estables.

___

#### 5\. `By.linkText()` — Per text exacte de l'enllaç

Localitza un element `<a>` pel seu text visible complet.

```php-template
<a href="/about">Sobre nosaltres</a>
```

```java
WebElement enllaç = driver.findElement(By.linkText("Sobre nosaltres"));
```

___

#### 6\. `By.partialLinkText()` — Per text parcial de l'enllaç

Com `linkText` però cerca per coincidència parcial.

```java
WebElement enllaç = driver.findElement(By.partialLinkText("Sobre"));
```

___

### 📊 Taula comparativa

## Estratègies de localització d'elements

| Estratègia | Velocitat | Fiabilitat | Quan utilitzar-la |
|----------------------|---------------|-------------|--------------------------------------------|
| `By.id()` | ⚡ Molt ràpida | ⭐⭐⭐ Alta | Sempre que l'element tingui `id` únic |
| `By.name()` | ⚡ Ràpida | ⭐⭐ Mitjana | Formularis amb atribut `name` |
| `By.cssSelector()` | ⚡ Ràpida | ⭐⭐⭐ Alta | Ús general, projectes professionals |
| `By.xpath()` | 🐢 Més lenta | ⭐⭐ Variable | DOMs complexos sense id ni classes fiables |
| `By.linkText()` | ⚡ Ràpida | ⭐⭐ Mitjana | Localitzar enllaços pel seu text exacte |
| `By.partialLinkText()` | ⚡ Ràpida | ⭐ Baixa | Quan el text de l'enllaç pot variar |

___

### 🔢 findElement vs findElements

```java
// Retorna UN sol WebElement (llança excepció si no el troba)
WebElement element = driver.findElement(By.cssSelector(".resultat"));

// Retorna una LLISTA de WebElements (retorna llista buida si no n'hi ha cap)
List<WebElement> elements = driver.findElements(By.cssSelector(".resultat"));
```

> 💡 **Tip:** Utilitza `findElements()` quan no saps quants elements hi ha a la pàgina, o per verificar que un element _no existeix_ (comprovant que la llista és buida).
