# NavigationTest.java- Navegación entre páginas | Odissea

---
Selenium WebDriver ofereix dues formes de navegar a una URL i un conjunt de mètodes per simular els botons del navegador: **enrere**, **endavant** i **recarregar**. Aquest test mostra totes aquestes opcions en acció.

___

### 📄 Codi del test

```scss
NavigationTest.java — Navegació entre pàginespublic class NavigationTest extends BaseTest {

    @Test
    public void testNavigation() throws Exception {

        driver.get("https://www.duckduckgo.com/");   // Navegar fins a DuckDuckGo
        Thread.sleep(2000);

        driver.navigate().to("https://www.google.com"); // Navegar fins a Google
        Thread.sleep(2000);

        driver.get("https://www.yahoo.com");            // Navegar fins a Yahoo
        Thread.sleep(2000);

        driver.navigate().back();    // ← Cap enrere (torna a Google)
        Thread.sleep(2000);

        driver.navigate().forward(); // → Cap endavant (torna a Yahoo)
        Thread.sleep(2000);

        driver.navigate().refresh(); // 🔄 Refresca Yahoo
        Thread.sleep(2000);
    }
}
```

___

### 🔍 Anàlisi línia a línia

#### `driver.get(url)` vs `driver.navigate().to(url)`

Tots dos naveguen a una URL, però hi ha una diferència subtil:

| Mètode | Comportament | Historial del navegador |
|---------------------------|---------------------------------------------------------------------|---------------------------------|
| `driver.get(url)` | Navega a la URL i **espera** que la pàgina carregui completament | ✅ Afegeix entrada a l'historial |
| `driver.navigate().to(url)` | Navega a la URL. Més flexible, accepta objectes `URL` a més de `String` | ✅ Afegeix entrada a l'historial |

> A la pràctica, per a tests senzills tots dos són equivalents. `driver.get()` és el més utilitzat per la seva simplicitat.

___

#### `driver.navigate().back()` — Botó enrere

Simula prémer el botó **← Enrere** del navegador. Navega a la pàgina anterior de l'historial de la sessió actual.

```scss
driver.navigate().back();
// Equivalent a prémer ← al navegador
```

#### `driver.navigate().forward()` — Botó endavant

Simula prémer el botó **→ Endavant** del navegador. Només funciona si prèviament s'ha navegat cap enrere.

```scss
driver.navigate().forward();
// Equivalent a prémer → al navegador
```

#### `driver.navigate().refresh()` — Recarregar pàgina

Simula prémer **F5** o el botó de recàrrega. Útil per a tests que verifiquen el comportament després de recarregar (ex: sessions, memòria cau).

```scss
driver.navigate().refresh();
// Equivalent a prémer F5
```

___

### ⏱️ Nota sobre `Thread.sleep()`

En aquest test s'utilitza `Thread.sleep(2000)` (espera de 2 segons) entre cada acció. Això es fa únicament amb **finalitats didàctiques** per poder veure visualment cada pas al navegador.

> ⚠️ **Important:** En tests reals de producció, `Thread.sleep()` és una mala pràctica. Fa els tests lents i inestables. L'alternativa correcta és utilitzar **Explicit Waits** amb `WebDriverWait`, que esperen només el temps necessari fins que es compleix una condició.

```bash
// ❌ Mala pràctica
Thread.sleep(2000);

// ✅ Bona pràctica
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.titleContains("Google"));
```
