# Què és Selenium WebDriver? | Odissea

---
**Selenium WebDriver** és una llibreria d'automatització de navegadors web de codi obert. Permet controlar un navegador real (Chrome, Firefox, Edge…) mitjançant codi, simulant exactament el que faria un usuari: fer clic, escriure text, navegar entre pàgines, etc.

S'utilitza principalment per a:

-   ✅ **Testing funcional** d'aplicacions web (proves end-to-end)
-   ✅ **Regressió automàtica** — verificar que res no es trenca en fer canvis
-   ✅ **Web scraping** i tasques d'automatització general

___

### 🏗️ Arquitectura de Selenium WebDriver

Selenium WebDriver es comunica amb el navegador a través d'un protocol estàndard anomenat **W3C WebDriver Protocol**. El flux és el següent:

```java
  El teu codi Java
       ↓
  Selenium WebDriver API
       ↓
  WebDriver Protocol (HTTP/JSON)
       ↓
  Driver executable (chromedriver / geckodriver)
       ↓
  Navegador real (Chrome / Firefox)
```

Cada navegador té el seu propi _driver_ executable que actua de pont:

-   🟡 **Chrome** → `chromedriver` [(descàrregues)](https://googlechromelabs.github.io/chrome-for-testing/)
-   🟠 **Firefox** → `geckodriver` [(descàrregues)](https://geckodriver.com/download/)
-   🔵 **Edge** → `msedgedriver` [(descàrregues)](https://developer.microsoft.com/es-es/microsoft-edge/tools/webdriver/?cs=3769934801&form=MA13LH#downloads)

### 🧩 Conceptes clau

## Què és Selenium WebDriver?

| Concepte | Descripció |
|------------------------|----------------------------------------------------------------------------------------------|
| **WebDriver** | Interfície principal de Selenium. Representa el navegador i exposa mètodes per controlar-lo. |
| **WebElement** | Representa un element HTML de la pàgina (botó, camp de text, enllaç…). |
| **Locator** | Estratègia per trobar un element: per `id`, `cssSelector`, `xpath`, etc. |
| **Driver executable** | Procés extern que tradueix les comandes de Selenium al llenguatge natiu del navegador. |
| **W3C WebDriver Protocol** | Estàndard HTTP/JSON que defineix com es comuniquen Selenium i el driver del navegador. |
| **TestNG** | Framework de testing en Java que organitza i executa els tests (similar a JUnit). |

___

> **💡 Nota important:** WebDriver _no_ és un plugin del navegador ni una extensió. És un procés extern que es comunica amb el navegador mitjançant la seva API nativa. Això el fa més fiable i proper al comportament real de l'usuari.
