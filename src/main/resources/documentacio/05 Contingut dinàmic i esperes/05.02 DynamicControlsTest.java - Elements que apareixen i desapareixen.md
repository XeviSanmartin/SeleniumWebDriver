# DynamicControlsTest.java - Elements que apareixen i desapareixen 

---
En aplicacions web modernes, els elements del DOM poden afegir-se o eliminar-se dinàmicament com a resposta a accions de l'usuari. Si Selenium intenta interactuar amb un element que encara no existeix (o que ja no existeix), llança una `NoSuchElementException`. La solució és usar **esperes explícites** amb `WebDriverWait` per esperar que el DOM assoleixi l'estat esperat. Treballa contra [the-internet.herokuapp.com/dynamic\_controls](http://the-internet.herokuapp.com/dynamic_controls).

___

### 📄 Codi del test

```java
DynamicControlsTest.java — Elements que apareixen i desapareixenpublic class DynamicControlsTest extends BaseTest {

    @Test
    public void titleTest() {
        driver.navigate().to("http://the-internet.herokuapp.com/dynamic_controls");

        // 1. Verificar que el checkbox és visible
        WebElement checkbox = driver.findElement(By.id("checkbox"));
        assertTrue(checkbox.isDisplayed());

        // 2. Marcar el checkbox
        checkbox.click();

        // 3. Prémer el botó "Remove" per eliminar el checkbox del DOM
        WebElement removeBtn = driver.findElement(
                By.xpath("//*[@id='checkbox-example']/button"));
        removeBtn.click();

        // 4. Capturar el spinner de càrrega (pot estar present o no)
        List<WebElement> loading = driver.findElements(
                By.xpath("//*[@id='loading']/img"));

        // 5. Esperar que aparegui l'indicador de càrrega O el missatge de resultat
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(20));
        wait.until(ExpectedConditions.or(
                ExpectedConditions.presenceOfElementLocated(By.id("loading")),
                ExpectedConditions.presenceOfElementLocated(By.id("message"))
        ));
    }
}
```

___

### 🔍 Anàlisi: l'ús de `ExpectedConditions.or()`

La part més interessant del test és l'ús de `or()` per gestionar dos estats possibles de la UI:

```less
wait.until(ExpectedConditions.or(
    ExpectedConditions.presenceOfElementLocated(By.id("loading")),  // Spinner carregant
    ExpectedConditions.presenceOfElementLocated(By.id("message"))   // Resultat final
));
```

Després de prémer "Remove", la pàgina pot estar en un d'aquests dos estats:

-   🔄 **Carregant:** apareix un spinner (`#loading`)
-   ✅ **Completat:** apareix el missatge de resultat (`#message`)

Amb `or()`, l'espera acaba tan aviat com **qualsevol de les dues condicions** es compleix. Això fa el test robust davant variacions de velocitat de xarxa o del servidor.

___

### 📊 Condicions d'espera més útils de `ExpectedConditions`

| Condició | Espera que... |
|-----------------------------------------|----------------------------------------------------------|
| `presenceOfElementLocated(locator)` | L'element existeixi al DOM (encara que no sigui visible) |
| `visibilityOfElementLocated(locator)` | L'element existeixi al DOM **i** sigui visible |
| `invisibilityOfElementLocated(locator)` | L'element no sigui visible (o no existeixi) |
| `elementToBeClickable(locator)` | L'element sigui visible i estigui habilitat per fer clic |
| `textToBePresentInElement(element, text)` | L'element contingui el text indicat |
| `or(condició1, condició2, ...)` | Almenys una de les condicions es compleixi |
| `and(condició1, condició2, ...)` | Totes les condicions es compleixin simultàniament |

___

### 🔑 `findElement` vs `findElements`

|  | `findElement()` | `findElements()` |
|--------------------------|-------------------------------|---------------------------------------------------|
| **Si no troba res** | Llança `NoSuchElementException` | Retorna una llista buida `[]` |
| **Ús típic** | Quan l'element ha d'existir | Quan l'element pot no existir (contingut dinàmic) |
| **Per verificar existència** | ❌ Requereix try/catch | ✅ `findElements(...).size() > 0` |
