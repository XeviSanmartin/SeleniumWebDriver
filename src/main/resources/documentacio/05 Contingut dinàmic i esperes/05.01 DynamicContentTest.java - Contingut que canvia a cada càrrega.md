# DynamicContentTest.java - Contingut que canvia a cada càrrega 

---
Algunes pàgines web mostren contingut diferent a cada recàrrega: imatges aleatòries, textos rotatius, dades en temps real. Aquest test verifica que el contingut **efectivament canvia** entre una càrrega i la següent, comparant els valors abans i després d'un `refresh()`. Treballa contra [the-internet.herokuapp.com/dynamic\_content](https://the-internet.herokuapp.com/dynamic_content).

___

### 📄 Codi del test

```java
DynamicContentTest.java — Contingut que canvia a cada càrregapublic class DynamicContentTest extends BaseTest {

    @Test
    public void testApp() throws InterruptedException {
        driver.navigate().to("https://the-internet.herokuapp.com/dynamic_content");

        // --- Càrrega inicial: capturar imatge i text del tercer bloc ---
        WebElement imatge1 = driver.findElement(
                By.xpath(".//*[@id='content']/div[3]/div[1]/img"));
        WebElement text1 = driver.findElement(
                By.xpath(".//*[@id='content']/div[3]/div[2]"));

        String urlImatge1  = imatge1.getAttribute("src");
        String texttotal1  = text1.getText();

        // --- Recarregar la pàgina ---
        driver.navigate().refresh();

        // --- Segona càrrega: capturar els mateixos elements de nou ---
        imatge1 = driver.findElement(
                By.xpath(".//*[@id='content']/div[3]/div[1]/img"));
        text1   = driver.findElement(
                By.xpath(".//*[@id='content']/div[3]/div[2]"));

        String urlImatge2  = imatge1.getAttribute("src");
        String texttotal2  = text1.getText();

        // --- Verificar que el contingut ha canviat ---
        assertFalse(urlImatge1.equals(urlImatge2));   // La imatge és diferent
        assertFalse(texttotal1.equals(texttotal2));   // El text és diferent
    }
}
```

___

### 🔍 Anàlisi pas a pas

#### 1\. Capturar l'estat inicial

```java
String urlImatge1 = imatge1.getAttribute("src");
String texttotal1 = text1.getText();
```

Abans de recarregar, guardem en variables l'atribut `src` de la imatge i el text del bloc. Aquests valors són la **referència** amb la qual compararem després del refresh.

#### 2\. Recarregar la pàgina

```scss
driver.navigate().refresh();
```

Equivalent a prémer F5 al navegador. El DOM es destrueix i es reconstrueix completament, de manera que les referències anteriors a `WebElement` queden invalidades (llançarien `StaleElementReferenceException` si s'usessin). Per això cal tornar a fer `findElement()` després del refresh.

> ⚠️ **StaleElementReferenceException:** Després de qualsevol navegació (`refresh()`, `back()`, `forward()`, `navigate().to()`), tots els `WebElement` capturats abans queden obsolets i cal relocalitzar-los.

#### 3\. Verificar que el contingut ha canviat

```scss
assertFalse(urlImatge1.equals(urlImatge2));
assertFalse(texttotal1.equals(texttotal2));
```

Usem `assertFalse(a.equals(b))` per verificar que els dos valors són **diferents**. Si fossin iguals, el test fallaria — indicant que la pàgina no està generant contingut dinàmic correctament.

___

### 📊 Mètodes de navegació de Selenium

| Mètode | Equivalent al navegador | Invalida WebElements |
|-----------------------------|----------------------------------------|----------------------|
| `driver.navigate().to(url)` | Escriure URL a la barra i prémer Enter | ✅ Sí |
| `driver.navigate().refresh()` | F5 / Recarregar pàgina | ✅ Sí |
| `driver.navigate().back()` | Botó Enrere del navegador | ✅ Sí |
| `driver.navigate().forward()` | Botó Endavant del navegador | ✅ Sí |
