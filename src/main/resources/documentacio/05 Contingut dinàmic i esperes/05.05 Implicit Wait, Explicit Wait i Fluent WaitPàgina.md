# Implicit Wait, Explicit Wait i Fluent Wait 

---

## ⏱️ Esperes a Selenium: Implicit Wait, Explicit Wait i Fluent Wait

Un dels problemes més freqüents en tests de Selenium són els errors per **condicions de cursa**: Selenium intenta interactuar amb un element abans que la pàgina hagi acabat de carregar-lo. La solució són les **estratègies d'espera**, que li diuen al driver que paussi l'execució fins que es compleixi una condició. Hi ha tres nivells, de menys a més control.

___

### 1️⃣ Implicit Wait — Espera implícita (global)

L'espera implícita configura un **temps màxim global** que Selenium esperarà a cada `findElement()` abans de llançar `NoSuchElementException`. Es configura una sola vegada i s'aplica a totes les cerques d'elements.

```java
// Configurar: esperar fins a 10 segons a cada findElement()
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

// A partir d'aquí, tots els findElement() esperaran fins a 10s automàticament
WebElement boto = driver.findElement(By.id("submit")); // Espera fins a 10s si encara no existeix
```

#### Com funciona internament

```less
// Sense implicit wait: falla immediatament si l'element no existeix
driver.findElement(By.id("element")); // → NoSuchElementException a l'instant

// Amb implicit wait de 10s: reintenta cada ~500ms durant 10 segons
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
driver.findElement(By.id("element")); // → Reintenta fins a 10s abans de fallar
```

|  | Implicit Wait |
|--------------|------------------------------------------------|
| **Abast** | Global — afecta tots els `findElement()` |
| **Configuració** | Una sola vegada a `@BeforeClass` o `@BeforeMethod` |
| **Condició** | Només espera que l'element _existeixi al DOM_ |
| **Problema** | No espera que sigui visible, clicable, etc. |

___

### 2️⃣ Explicit Wait — Espera explícita amb `WebDriverWait`

L'espera explícita espera que es compleixi una **condició específica** abans de continuar. És molt més potent que la implícita perquè permet definir exactament quin estat ha de tenir l'element (visible, clicable, amb cert text, etc.).

```java
⏱️ Esperes a Selenium: Implicit Wait, Explicit Wait i Fluent Wait// Crear l'objecte d'espera amb timeout de 10 segons
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));

// Esperar que l'element sigui clicable
WebElement boto = wait.until(
    ExpectedConditions.elementToBeClickable(By.id("submit"))
);

// Esperar que aparegui un text concret
wait.until(
    ExpectedConditions.textToBePresentInElementLocated(By.id("missatge"), "Èxit")
);

// Esperar que un element desaparegui
wait.until(
    ExpectedConditions.invisibilityOfElementLocated(By.id("spinner"))
);

// Esperar que hi hagi exactament 2 finestres obertes
wait.until(ExpectedConditions.numberOfWindowsToBe(2));

// Combinar condicions amb or()
wait.until(ExpectedConditions.or(
    ExpectedConditions.presenceOfElementLocated(By.id("loading")),
    ExpectedConditions.presenceOfElementLocated(By.id("message"))
));
```

#### Com funciona internament

```java
// WebDriverWait reintenta la condició cada 500ms fins que:
// a) La condició retorna true → continua el test ✅
// b) S'esgota el timeout → llança TimeoutException ❌

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
//                                              ↑ timeout màxim
// Interval de reintent per defecte: 500ms
```

|  | Explicit Wait |
|----------------------|----------------------------------------------------------------|
| **Abast** | Local — només on s'usa |
| **Condició** | Qualsevol `ExpectedConditions.*` o lambda personalitzada |
| **Interval de reintent** | 500ms per defecte |
| **Si falla** | Llança `TimeoutException` |
| **Recomanat per a** | ✅ La majoria de casos — és l'estratègia més usada en producció |

___

### 3️⃣ Fluent Wait — Espera amb control total

`FluentWait` és la versió més configurable. Permet definir el timeout, l'interval de reintent, quines excepcions ignorar durant l'espera, i usar condicions personalitzades amb lambdas.

```java
import org.openqa.selenium.support.ui.FluentWait;
import java.time.Duration;
import java.util.NoSuchElementException;

// Configurar FluentWait
FluentWait<WebDriver> fluentWait = new FluentWait<>(driver)
    .withTimeout(Duration.ofSeconds(30))          // Timeout màxim: 30 segons
    .pollingEvery(Duration.ofMillis(500))          // Reintentar cada 500ms
    .ignoring(NoSuchElementException.class);       // Ignorar aquesta excepció durant l'espera

// Usar amb ExpectedConditions
WebElement element = fluentWait.until(
    ExpectedConditions.elementToBeClickable(By.id("submit"))
);

// Usar amb lambda personalitzada
WebElement resultat = fluentWait.until(driver -> {
    WebElement el = driver.findElement(By.id("resultat"));
    return el.isDisplayed() ? el : null; // Retorna null per continuar esperant
});
```

___

### 📊 Comparativa de les tres estratègies

|  | Implicit Wait | Explicit Wait | Fluent Wait |
|-----------------------|------------------------|-----------------------------|--------------------------------|
| **Abast** | Global | Local | Local |
| **Condició** | Només presència al DOM | Qualsevol ExpectedCondition | Qualsevol condició + lambda |
| **Interval configurable** | ❌ No | ❌ Fix (500ms) | ✅ Sí |
| **Ignorar excepcions** | ❌ No | ❌ No | ✅ Sí |
| **Complexitat** | ⭐ Baixa | ⭐⭐ Mitjana | ⭐⭐⭐ Alta |
| **Ús recomanat** | Setup inicial bàsic | ✅ La majoria de tests | Casos complexos o intermitents |

> ⚠️ **Mai barregis Implicit Wait i Explicit Wait** en el mateix test. Poden interferir entre si i produir comportaments imprevisibles, com timeouts que duren el doble del previst.

___

### ❌ `Thread.sleep()` — Per què no usar-lo

```java
// ❌ MAL: espera fixa de 3 segons, sempre, encara que l'element ja estigui llest
Thread.sleep(3000);

// ✅ BÉ: espera només el necessari, fins a 10 segons com a màxim
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.elementToBeClickable(By.id("submit")));
```

|  | `Thread.sleep()` ❌ | `WebDriverWait` ✅ |
|--------------------|------------------------------------------|---------------------------------------|
| **Temps d'espera** | Sempre el temps complet | Només el necessari |
| **Velocitat del test** | Lent (espera encara que no calgui) | Ràpid (continua quan està llest) |
| **Fiabilitat** | Fràgil (pot fallar si la xarxa és lenta) | Robust (s'adapta a la velocitat real) |
| **Ús acceptable** | Només en demos o debugging puntual | Tests de producció |
