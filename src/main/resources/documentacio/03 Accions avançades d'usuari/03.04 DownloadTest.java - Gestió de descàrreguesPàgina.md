# DownloadTest.java - Gestió de descàrregues | Odissea

---
Gestionar descàrregues a Selenium requereix una configuració especial del navegador: cal indicar-li **on guardar els fitxers** i que ho faci **sense mostrar el diàleg de confirmació**. Això s'aconsegueix mitjançant les preferències del navegador abans d'instanciar el driver. Aquest test treballa contra [the-internet.herokuapp.com/download](http://the-internet.herokuapp.com/download).

___

### 📄 Codi del test

```java
DownloadTest.java — Gestió de descàrreguespublic class DownloadTest extends BaseTest {

    File folder; // Carpeta temporal on es guardaran les descàrregues

    @BeforeMethod
    public void setUp() {
        // Crear una carpeta temporal amb nom únic (UUID)
        folder = new File("src" + File.separator + "main"
            + File.separator + "resources"
            + File.separator + UUID.randomUUID().toString());

        // --- Configuració de Chrome ---
        String downloadFilepath = folder.getAbsolutePath();
        HashMap<String, Object> chromePrefs = new HashMap<>();
        chromePrefs.put("profile.default_content_settings.popups", 0);
        chromePrefs.put("download.default_directory", downloadFilepath);

        ChromeOptions options = new ChromeOptions();
        options.setExperimentalOption("prefs", chromePrefs);
        options.setAcceptInsecureCerts(true);

        driver = new ChromeDriver(options);
    }

    @AfterMethod
    public void tearDown() {
        driver.quit();
    }

    @Test
    public void download() throws Exception {
        // 1. Crear la carpeta de descàrregues
        folder.mkdir();

        // 2. Navegar a la pàgina de descàrregues
        driver.get("http://the-internet.herokuapp.com/download");
        Thread.sleep(2000);

        // 3. Localitzar el primer enllaç de descàrrega i fer clic
        Actions actions = new Actions(driver);
        WebElement link = driver.findElement(
            By.xpath("/html[1]/body[1]/div[2]/div[1]/div[1]/a[1]"));
        actions.dragAndDrop(link, link).build().perform();
        Thread.sleep(2000);

        // 4. Verificar que la carpeta no està buida (s'ha descarregat alguna cosa)
        File[] listOfFiles = folder.listFiles();
        assertThat(listOfFiles.length, is(not(0)));

        // 5. Verificar que els fitxers descarregats no estan buits
        for (File file : listOfFiles) {
            assertThat(file.length(), is(not((long) 0)));
        }

        // 6. Netejar — esborrar els fitxers i la carpeta temporal
        String[] entries = folder.list();
        for (String s : entries) {
            new File(folder.getPath(), s).delete();
        }
        folder.delete();
    }
}
```

___

### 🔍 Anàlisi: les tres parts clau

#### Part 1 — Configurar Chrome per descarregar sense diàlegs

La configuració es fa a `@BeforeMethod`, abans que el driver arrenqui. S'utilitzen les **preferències experimentals de Chrome** per:

-   Desactivar el popup de confirmació de descàrrega (`popups = 0`)
-   Establir la carpeta de destí de les descàrregues (`download.default_directory`)

```javascript
HashMap<String, Object> chromePrefs = new HashMap<>();
chromePrefs.put("profile.default_content_settings.popups", 0);
chromePrefs.put("download.default_directory", folder.getAbsolutePath());

ChromeOptions options = new ChromeOptions();
options.setExperimentalOption("prefs", chromePrefs);
driver = new ChromeDriver(options);
```

> 💡 **Nota:** Aquest test sobreescriu el `setUp()` heretat de `BaseTest` usant `@BeforeMethod` en lloc de `@BeforeClass`, per poder configurar les opcions de Chrome abans d'instanciar el driver.

___

#### Part 2 — Carpeta temporal amb UUID

```ini
folder = new File("src/main/resources/" + UUID.randomUUID().toString());
```

S'utilitza `UUID.randomUUID()` per generar un nom de carpeta únic a cada execució. Això evita conflictes si el test s'executa diverses vegades en paral·lel o en successió ràpida.

___

#### Part 3 — Verificació i neteja

```scss
// Verificar que s'ha descarregat almenys un fitxer
assertThat(listOfFiles.length, is(not(0)));

// Verificar que el fitxer no està buit (0 bytes)
assertThat(file.length(), is(not((long) 0)));

// Netejar la carpeta temporal en acabar
folder.delete();
```

La verificació no comprova el nom del fitxer (pot variar), sinó que **existeix almenys un fitxer** i que **té contingut**. Al final el test neteja la carpeta per no deixar residus al projecte.

### 🦊 Configuració equivalent per a Firefox (comentada al test)

```java
FirefoxProfile profile = new FirefoxProfile();

// Tipus MIME que es descarreguen sense preguntar
profile.setPreference(
    "browser.helperApps.neverAsk.saveToDisk",
    "application/octet-stream;application/csv;text/csv");

// Desactivar el gestor de descàrregues
profile.setPreference("browser.download.manager.showWhenStarting", false);

// Carpeta de destí (2 = carpeta personalitzada)
profile.setPreference("browser.download.folderList", 2);
profile.setPreference("browser.download.dir", folder.getAbsolutePath());

FirefoxOptions options = new FirefoxOptions();
options.setProfile(profile);
driver = new FirefoxDriver(options);
```

___

### 📊 Comparativa de configuració de descàrregues per navegador

|  | Chrome | Firefox |
|-----------------|---------------------------------------------|-------------------------------------------------------------|
| **Mecanisme** | `ChromeOptions + experimentalOption("prefs")` | `FirefoxProfile` |
| **Carpeta destí** | `"download.default_directory"` | `"browser.download.dir"` |
| **Sense diàleg** | `"profile.default_content_settings.popups", 0` | `"browser.helperApps.neverAsk.saveToDisk"` amb els tipus MIME |
| **Quan configurar** | Abans d'instanciar `ChromeDriver` | Abans d'instanciar `FirefoxDriver` |

___

### ▶️ Com executar aquest test

```bash
mvn test -Dtest=DownloadTest
```

> ℹ️ Aquest test només funciona amb Chrome (la configuració de Firefox està comentada al codi). No cal passar `-Dbrowser` ja que el test instancia el driver directament.
